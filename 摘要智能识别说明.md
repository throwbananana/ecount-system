# 摘要智能识别功能说明

## 功能概述

**摘要智能识别**是亿看智能识别系统的核心创新功能，能够自动分析Excel文件中的"摘要"字段内容，智能识别业务类型、往来单位、科目编码、金额、日期等信息，并自动填充到一般凭证模板的相应字段中。

### 核心价值

- ✅ **减少手工映射**：自动识别字段，无需逐列手动映射
- ✅ **提高准确性**：基于业务规则自动匹配科目和类型
- ✅ **节省时间**：批量处理时效率提升80%以上
- ✅ **灵活可扩展**：支持自定义识别规则

---

## 启用方式

在Excel凭证转换标签页中：

1. 选择Excel文件和工作表
2. 确保 **"启用摘要智能识别（自动填充字段）"** 复选框已勾选（默认启用）
3. 映射"摘要"列到原始数据中的摘要字段
4. 点击"自动识别匹配"或"开始转换并导出"

---

## 识别能力

### 1. 业务类型识别

根据摘要中的关键词，自动识别业务类型并填充相应字段。

| 业务类型 | 关键词 | 自动填充字段 |
|---------|--------|-------------|
| **销售** | 销售、销货、售出、卖出 | 摘要编码: 01<br>类型: 1(出)<br>科目: 1122(应收账款) |
| **销售退回** | 销售退回、退货、退款 | 摘要编码: 02<br>类型: 2(入)<br>科目: 1122 |
| **采购** | 采购、购买、进货、买入 | 摘要编码: 03<br>类型: 2(入)<br>科目: 2202(应付账款) |
| **采购退回** | 采购退回 | 摘要编码: 04<br>类型: 1(出)<br>科目: 2202 |
| **收款** | 收到、收款、回款、收汇 | 摘要编码: 05<br>类型: 1(出)<br>科目: 1002(银行存款) |
| **付款** | 支付、付款、付汇、转账 | 摘要编码: 06<br>类型: 2(入)<br>科目: 1002 |
| **运费** | 运费、运输费、物流费 | 摘要编码: 07<br>类型: 1(出)<br>科目: 6401(销售费用-运费) |
| **办公费** | 办公费、办公用品、文具 | 摘要编码: 08<br>类型: 1(出)<br>科目: 6602(管理费用-办公费) |
| **工资** | 工资、薪酬、薪资 | 摘要编码: 09<br>类型: 2(入)<br>科目: 2211(应付职工薪酬) |
| **入库** | 入库、验收入库、采购入库 | 摘要编码: 10<br>类型: 1(出)<br>科目: 1405(库存商品) |
| **出库** | 出库、发货、领用 | 摘要编码: 11<br>类型: 2(入)<br>科目: 1405 |

### 2. 往来单位识别

从摘要中识别往来单位名称并自动匹配编码（基于数据库）。

**示例**：
- 摘要：`销售商品给ABEL CAPOTE`
- 识别结果：
  - 往来单位编码: `H0405`
  - 往来单位名: `ABEL CAPOTE`

**支持的匹配方式**：
1. **精确匹配**：完全匹配数据库中的名称
2. **简化名称匹配**：去除括号等符号后匹配
3. **公司名模式识别**：
   - 英文公司：`XXX CO`, `XXX LTD`, `XXX CORP`
   - 中文公司：`XX公司`, `XX商店`, `XX贸易`

### 3. 科目编码识别

识别摘要中明确提到的科目或从业务类型推断。

**识别方式**：
1. **直接引用**：摘要中包含 `[1001]` 或 `1001`
2. **科目名称**：摘要中包含 `库存现金`、`银行存款` 等
3. **业务推断**：从业务类型自动推断科目

**示例**：
- `收到货款存入银行` → 科目编码: `1002`（银行存款）
- `支付工资` → 科目编码: `2211`（应付职工薪酬）

### 4. 部门识别

识别部门名称或关键词。

**支持识别**：
- 数据库中的部门名称
- 关键词：`巴拿马` → 部门编码 `10001`
- 关键词：`中国` → 部门编码 `10002`

**示例**：
- `发放工资给巴拿马部门` → 部门: `10001`

### 5. 金额识别

从摘要中提取金额数字。

**支持格式**：
- `500元` → 500.0
- `1000美元` → 1000.0
- `USD 2000` → 2000.0
- `¥3,500` → 3500.0
- `$4500.50` → 4500.5
- `5万元` → 50000.0

**示例**：
- `采购原材料入库，金额1000元` → 金额: 1000.0

### 6. 日期识别

从摘要中提取日期。

**支持格式**：
- `2025-01-15` → 20250115
- `2025/01/15` → 20250115
- `2025年1月15日` → 20250115
- `20250115` → 20250115

**示例**：
- `2025-01-15采购退回，金额800元` → 凭证日期: 20250115

| 销售鞋类商品给ABEL CAPOTE，金额500美元 |

|-----|---|

| 摘要编码 | 01 |
| 类型 | 1 |
| 科目编码 | 1122 |
| 往来单位编码 | H0405 |
| 往来单位名 | ABEL CAPOTE |
| 金额 | 500.0 |

### 场景2：部分字段已映射，智能补充其他字段

**原始Excel**：
| 日期 | 摘要 | 金额 |
|------|------|------|
| 2025-01-15 | 采购原材料入库 | 1000 |

**用户映射**：
- 凭证日期 ← 日期
- 摘要 ← 摘要
- 金额 ← 金额

**智能补充**（自动填充）：
- 摘要编码 ← 03（识别出"采购"）
- 类型 ← 2（识别出"采购"为入账）
- 科目编码 ← 2202（应付账款）

### 场景3：复杂摘要综合识别

**摘要**：`2025-01-20收到ADAIN BENITEZ RODRIGUEZ的货款2000美元存入巴拿马银行账户`

**识别结果**：
- 凭证日期: 20250120
- 摘要编码: 05
- 类型: 1
- 科目编码: 1002
- 往来单位编码: TE963
- 往来单位名: ADAIN BENITEZ RODRIGUEZ
- 金额: 2000.0
- 部门: 10001

## 优先级规则

当同一字段有多个来源时，优先级为：

1. **用户手动映射** - 最高优先级
2. **智能识别结果** - 当没有手动映射时使用
3. **空值** - 都没有时留空

**示例**：
- 用户将"金额"列映射到原始数据的"amt"字段 → 使用"amt"的值
- 用户未映射"科目编码"字段 → 使用智能识别的科目
- 摘要中也未识别出科目 → 该字段留空

## 自定义识别规则

可以通过修改 `summary_intelligence.py` 来自定义识别规则。

### 添加新的业务类型

编辑 `_init_recognition_rules()` 方法中的 `business_type_rules` 字典：

```python
self.business_type_rules = {
    # ... 现有规则 ...

    "新业务类型": {
        "keywords": ["关键词1", "关键词2"],  # 识别关键词
        "account": "XXXX",                    # 默认科目编码
        "type": "1",                          # 类型(1=出, 2=入, 3=借, 4=贷)
        "summary_code": "XX"                  # 摘要编码
    },
}
```

### 添加部门关键词

编辑 `_recognize_department()` 方法中的 `dept_keywords`：

```python
dept_keywords = {
    "巴拿马": "10001",
    "中国": "10002",
    "新部门关键词": "新部门编码",
}
```

### 扩展金额识别

在 `_recognize_amount()` 方法中添加新的匹配模式：

```python
patterns = [
    # ... 现有模式 ...
    r'新的金额模式',
]
```

## 注意事项

### 1. 数据库依赖

智能识别依赖基础数据库的内容：
- 往来单位识别需要 `business_partner` 表
- 科目识别需要 `account_subject` 表
- 部门识别需要 `department` 表

**建议**：
- 导入数据前先确保基础数据已导入
- 定期更新基础数据以提高识别准确率

### 2. 中文分词

识别器使用简单的关键词匹配，对于复杂的自然语言可能识别不准确。

**最佳实践**：
- 摘要中尽量包含明确的关键词
- 避免过于口语化的表述
- 金额、日期使用标准格式

### 3. 多义性处理

当摘要中出现冲突信息时，使用第一个匹配的规则。

**示例**：
- `采购后销售` → 识别为"采购"（先匹配到"采购"关键词）

### 4. 性能考虑

- 智能识别会增加一定的处理时间
- 对于大批量数据（>10000行），建议分批处理
- 可以通过取消勾选"启用摘要智能识别"来禁用此功能

## 常见问题

### Q1: 为什么某些字段没有被识别到？
**A**: 可能原因：
1. 摘要中没有包含相关关键词
2. 基础数据库中缺少对应记录
3. 识别规则未覆盖该场景

**解决方案**：
- 检查摘要内容是否包含关键信息
- 确认基础数据已正确导入
- 自定义添加识别规则

### Q2: 识别结果不准确怎么办？
**A**:
- 检查是否有用户手动映射覆盖了智能识别结果
- 查看测试脚本输出，确认识别逻辑
- 调整摘要内容或自定义识别规则

### Q3: 可以同时使用手动映射和智能识别吗？
**A**: 可以！系统采用"手动优先"原则：
- 已手动映射的字段使用映射值
- 未映射的字段尝试智能识别
- 两者可完美配合

### Q4: 识别到的往来单位不对怎么办？
**A**:
- 检查数据库中是否有重名的往来单位
- 在摘要中使用完整准确的单位名称
- 手动映射"往来单位编码"字段

## 技术架构

### 核心组件

```
SummaryIntelligence (摘要智能识别器)
├── _init_recognition_rules()    # 初始化业务识别规则
├── _load_base_data_cache()     # 加载基础数据到缓存
├── recognize()                  # 主识别函数
│   ├── _recognize_business_type()  # 识别业务类型
│   ├── _recognize_partner()        # 识别往来单位
│   ├── _recognize_account()        # 识别科目
│   ├── _recognize_department()     # 识别部门
│   ├── _recognize_amount()         # 识别金额
│   └── _recognize_date()           # 识别日期
└── batch_recognize()            # 批量识别
```

### 识别流程

```
1. 输入摘要文本
   ↓
2. 业务类型识别（关键词匹配）
   ↓
3. 往来单位识别（数据库匹配 + 模式识别）
   ↓
4. 科目编码识别（直接引用 + 名称匹配 + 业务推断）
   ↓
5. 部门识别（数据库匹配 + 关键词）
   ↓
6. 金额识别（正则表达式提取）
   ↓
7. 日期识别（多格式匹配）
   ↓
8. 返回识别结果字典
```
