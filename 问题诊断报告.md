## 🔍 根本原因分析

### 原因1：使用了错误的模板/列名

**错误的导出结果.xlsx：**
```
列1: 日期
列2: 序号
列3: 科目编码     ← 空白
列4: 往来单位编码  ← 空白
列5: 往来单位名    ← 空白
列6: 金额         ← 空白
列7: 摘要名       ← ⚠️ 错误！应该是"摘要"
列8: 部门         ← 空白
```

**问题：**
- 列7的名称是 **"摘要名"** 而不是 **"摘要"**
- 智能识别器只识别字段名为 **"摘要"** 的列
- 因为字段名不匹配，智能识别功能完全无效

**正确的模板应该是：**
```
列1: 日期
列2: 序号
列3: 摘要         ← ✓ 正确的字段名
列4: 科目编码
列5: 对方科目
列6: 默认账户
...
```

### 原因2：没有启用智能识别功能

转换时需要勾选：
```
☐ 启用摘要智能识别（自动填充字段）  ← 你可能没有勾选
```

### 原因3：列映射不正确

源数据有5列：
- 发生日期
- **摘要** ← 正确的源列名
- 收入明细金额
- 支出明细金额
- 余额

但转换时可能：
- "摘要"列没有正确映射
- 或者映射到了错误的目标字段（"摘要名"）

## 🎯 关键要点

### ⚠️ 为什么智能识别没有工作

智能识别器在代码中是这样工作的：

```python
# 第1步：查找"摘要"列
summary_col = mapping.get("摘要")  # ← 只识别名为"摘要"的字段

# 第2步：如果找到"摘要"列，才进行智能识别
if summary_col:
    summary_value = src_row[summary_col]
    smart_data = recognizer.recognize(summary_value)
    # 自动填充：科目编码、部门、金额等
```

**如果列名是"摘要名"而不是"摘要"：**
- `mapping.get("摘要")` 返回 None
- 智能识别代码根本不会执行
- 所有字段保持为空

## ❓ 常见问题

### Q1: 为什么"对方科目"和"默认账户"还是空的？

A: 智能识别器目前只自动填充：
- 科目编码 ✓
- 部门 ✓
- 金额 ✓
- 往来单位编码/名 ✓

"对方科目"和"默认账户"需要：
- 方法1：在源数据中添加这两列
- 方法2：使用"默认值"功能批量填充固定值
- 方法3：转换后手动填写

### Q2: 能否让系统自动填充"对方科目"？

A: 可以！如果你有具体的规则（如：对方科目总是固定值，或基于科目编码的对应关系），告诉我，我可以扩展智能识别功能。

### Q3: 为什么有些摘要识别不到科目编码？

A: 可能原因：
1. 该摘要在智能识别缓存中没有记录
2. 摘要内容太特殊，规则无法匹配

解决方法：
- 在GUI中进入"基础数据管理" → "智能识别缓存"
- 手动添加摘要 → 科目编码的映射
- 下次转换时会自动使用
