# 多字段智能识别和默认值功能说明

## 功能概述

系统现在支持从**多个字段**进行智能识别，并可以为常用字段设置**默认值**，进一步减少手动映射工作量。

### 核心改进

1. **多字段识别**：不仅从"摘要"识别，还从"日期"、"金额"、"汇率"等字段自动提取
2. **默认值设置**：为部门、外币金额、汇率等字段设置固定默认值
3. **优先级控制**：清晰的字段值来源优先级

---

## 一、多字段智能识别

### 支持的识别字段

| 源字段 | 识别目标字段 | 支持的列名变体 |
|--------|------------|--------------|
| **日期字段** | 凭证日期 | 日期、凭证日期、date、Date |
| **金额字段** | 金额 | 金额、amount、Amount、amt |
| **汇率字段** | 汇率 | 汇率、rate、Rate、exchange_rate |
| **外币字段** | 外币金额 | 外币金额、外币、foreign_amount |
| **摘要字段** | 多个字段 | 摘要（详见摘要智能识别说明） |

### 识别示例

#### 示例1：从多个字段提取

**原始Excel数据**：
```
| 日期       | 摘要                          | 金额   | 汇率 |
|-----------|------------------------------|--------|------|
| 2025-01-15| 销售鞋类商品给ABEL CAPOTE     | 500.50 | 6.5  |
```

**识别结果**：
- 凭证日期: `20250115` （从"日期"字段）
- 金额: `500.5` （从"金额"字段）
- 汇率: `6.5` （从"汇率"字段）
- 摘要编码: `01` （从"摘要"识别：销售）
- 类型: `1` （从"摘要"识别：出）
- 科目编码: `1122` （从"摘要"识别：应收账款）
- 往来单位: `ABEL CAPOTE` （从"摘要"识别）

#### 示例2：字段名变体支持

系统会自动匹配不同的列名：

```
| date       | amt   | rate |
|-----------|-------|------|
| 2025-01-15| 300   | 6.8  |
```

会正确识别为：
- 凭证日期: `20250115`
- 金额: `300.0`
- 汇率: `6.8`

### 日期格式支持

支持多种日期格式：
- `YYYYMMDD`：20250115
- `YYYY-MM-DD`：2025-01-15
- `YYYY/MM/DD`：2025/01/15
- `YYYY年MM月DD日`：2025年1月15日

### 金额格式支持

支持多种金额格式：
- 纯数字：`500.50`
- 带单位：`500元`、`500美元`、`USD 500`
- 带千位分隔符：`1,000.50`
- 中文单位：`5万元` → `50000`

### 汇率格式支持

- 小数格式：`6.5`、`6.8012`
- 整数格式：`1`
- 带千位分隔符：`1,234.56`
- 合理范围：0.0001 ~ 10000

## 二、默认值设置

### 功能入口

在"Excel凭证转换"标签页中，点击**"设置默认值..."**按钮。

### 可设置默认值的字段

| 字段名 | 用途 | 示例值 |
|--------|------|--------|
| **部门** | 设置默认部门编码 | `10001`（巴拿马） |
| **外币金额** | 设置默认外币金额 | `0` |
| **汇率** | 设置默认汇率 | `1` 或 `6.5` |
| **序号** | 设置默认序号 | `1` |
| **会计凭证No.** | 设置默认凭证号 | `PZ001` |

### 使用方法

1. 点击"设置默认值..."按钮
2. 在弹出的对话框中输入各字段的默认值
3. 点击"保存"按钮
4. 默认值会在转换时自动应用

### 默认值对话框

```
┌────────────────────────────────────────────┐
│ 设置默认值                                  │
├────────────────────────────────────────────┤
│ 为未映射或未识别的字段设置默认值：          │
│                                            │
│ 部门:        [10001       ]  部门编码...   │
│ 外币金额:     [0          ]  外币金额...   │
│ 汇率:        [1          ]  汇率，例如...  │
│ 序号:        [1          ]  序号，例如...  │
│ 会计凭证No.: [           ]  会计凭证号     │
│                                            │
│                      [取消] [清空] [保存]  │
└────────────────────────────────────────────┘
```

### 默认值应用规则

默认值会在以下情况使用：
- ✅ 字段**未手动映射**
- ✅ 字段**未从原始数据识别到**
- ✅ 字段**未从摘要识别到**

**不会覆盖**：
- ❌ 用户手动映射的字段
- ❌ 从原始数据提取的字段
- ❌ 从摘要识别的字段

## 三、字段值优先级

当同一个字段有多个来源时，系统按以下优先级决定使用哪个值：

```
1. 手动映射（最高优先级）
   ↓
2. 原始数据字段（日期、金额、汇率等直接字段）
   ↓
3. 摘要智能识别
   ↓
4. 默认值（最低优先级）
```

### 优先级示例

**场景**：日期字段同时存在多个来源

**原始数据**：
```
| 日期       | 摘要                          | 金额  |
|-----------|------------------------------|-------|
| 2025-01-20| 2025-01-25采购退回，金额800元 | 1000  |
```

**默认值设置**：
- 部门: `10001`
- 外币金额: `0`

**识别结果**：
- **凭证日期**: `20250120` ← 来自原始数据"日期"字段（优先级2）
- **金额**: `1000` ← 来自原始数据"金额"字段（优先级2）
- **摘要编码**: `03` ← 来自摘要识别"采购"（优先级3）
- **部门**: `10001` ← 来自默认值（优先级4）

**说明**：
- 虽然摘要中也包含日期`2025-01-25`，但优先使用原始数据字段的`2025-01-20`
- 虽然摘要中也包含金额`800元`，但优先使用原始数据字段的`1000`

## 四、实际使用场景

### 场景1：Excel有完整字段

**原始Excel**：
```
| 日期       | 客户        | 摘要        | 金额  | 汇率 |
|-----------|-----------|------------|-------|------|
| 2025-01-15| ABEL CAPOTE| 销售鞋类   | 500   | 6.5  |
```

**操作**：
1. 映射：凭证日期 ← 日期、往来单位名 ← 客户、摘要 ← 摘要、金额 ← 金额、汇率 ← 汇率
2. 启用智能识别
3. 转换

**结果**：
- 凭证日期、金额、汇率：从映射的字段直接提取
- 摘要编码、类型、科目编码：从摘要智能识别
- 部门、外币金额：使用默认值

### 场景2：Excel只有部分字段

**原始Excel**：
```
| 日期       | 摘要                              |
|-----------|----------------------------------|
| 2025-01-15| 收到货款2000美元，汇率6.8         |
```

**操作**：
1. 映射：凭证日期 ← 日期、摘要 ← 摘要
2. 设置默认值：部门=10001、外币金额=0、汇率=1
3. 启用智能识别
4. 转换

**结果**：
- 凭证日期：从"日期"字段
- 金额：从摘要识别 `2000`
- 汇率：从摘要识别 `6.8`（覆盖默认值）
- 业务类型：从摘要识别（收款）
- 部门：使用默认值 `10001`
- 外币金额：使用默认值 `0`

### 场景3：批量数据统一设置

**背景**：导入100行销售数据，都是同一个部门，外币交易

**原始Excel**：
```
| 日期       | 摘要                    | 金额  |
|-----------|------------------------|-------|
| 2025-01-15| 销售给客户A            | 500   |
| 2025-01-16| 销售给客户B            | 800   |
...（共100行）
```

**操作**：
1. 设置默认值：
   - 部门: `10001`
   - 汇率: `6.5`
   - 外币金额: `0`
2. 映射字段并启用智能识别
3. 一键转换

**优势**：
- 无需为每一行手动填写部门
- 无需为每一行手动填写汇率
- 大幅提高批量处理效率

## 五、配置建议

### 常用默认值配置

#### 配置1：巴拿马业务（外币）
```
部门: 10001
外币金额: 0
汇率: 6.5
```

#### 配置2：中国业务（本币）
```
部门: 10002
外币金额: 0
汇率: 1
```

#### 配置3：通用配置
```
外币金额: 0
汇率: 1
序号: 1
```

### 使用技巧

1. **按批次调整默认值**
   - 导入巴拿马数据前，设置部门=10001
   - 导入中国数据前，设置部门=10002
   - 避免后期逐行修改

2. **结合手动映射**
   - 如果Excel中有"部门"列，直接映射（优先级更高）
   - 如果Excel中没有，使用默认值
   - 灵活组合

3. **外币处理**
   - 如果是外币业务且Excel有"外币金额"列，映射该列
   - 如果没有，设置默认值"0"
   - 汇率通常设置为当日汇率

4. **清空默认值**
   - 处理不同性质数据时，点击"清空"按钮
   - 避免不相关的默认值影响结果

## 六、优势总结

### 1. 减少手动工作量

**之前**：
- Excel中没有"部门"列 → 导出后逐行填写
- Excel中没有"汇率"列 → 导出后逐行填写
- 需要大量手动操作

**现在**：
- 设置一次默认值 → 自动应用到所有行
- 大幅减少重复工作

### 2. 提高准确性

**之前**：
- 只从摘要识别，如果摘要中包含多个日期/金额，可能识别错误

**现在**：
- 优先使用Excel原始字段（日期、金额、汇率）
- 更准确，更可靠

### 3. 灵活性更强

- 可以完全依赖智能识别
- 可以部分映射 + 部分识别
- 可以映射 + 识别 + 默认值混合使用
- 适应各种Excel格式

### 4. 批量处理更高效

- 统一设置默认值
- 一次转换完成
- 无需后期逐行修改

---

## 七、注意事项

### 1. 优先级理解

- 手动映射 > 原始数据字段 > 摘要识别 > 默认值
- 高优先级的值会覆盖低优先级的值

### 2. 字段名匹配

- 系统会尝试匹配多种字段名变体
- 如果Excel列名太特殊，可能识别不到
- 建议使用常见列名：日期、金额、汇率等

### 3. 默认值适用范围

- 默认值对**所有行**生效
- 如果数据包含不同部门/不同汇率，不建议设置默认值
- 或者分批导入，分别设置

### 4. 数据验证

- 使用预处理预览窗口查看识别结果
- 确认默认值是否正确应用
- 确认字段优先级是否符合预期

## 九、常见问题

### Q1: 默认值不生效？

**可能原因**：
1. 该字段已手动映射（映射优先级更高）
2. 该字段从原始数据或摘要识别到值（识别优先级更高）

**解决方案**：
- 查看预处理预览窗口，确认字段来源
- 如需强制使用默认值，取消该字段的映射

### Q2: 从原始数据识别的字段不对？

**可能原因**：
- Excel列名不匹配（例如列名是"Date1"而非"日期"）

**解决方案**：
- 手动映射该字段（手动映射优先级最高）
- 或修改Excel列名为标准名称

### Q3: 设置默认值后能否修改？

**可以**：
- 随时点击"设置默认值..."重新设置
- 点击"清空"按钮清除所有默认值
- 修改后会立即生效

### Q4: 默认值会保存吗？

**不会持久化保存**：
- 默认值仅在当前程序运行期间有效
- 关闭程序后默认值会清空
- 下次运行需要重新设置

**建议**：
- 如果经常使用相同默认值，可以在代码中设置初始默认值
- 或记录常用默认值配置，每次手动设置
